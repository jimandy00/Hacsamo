
Gesture 부분
----------------------------------------------------------------

사죄의 말씀:
여러분,, 밥벌이가 바빠서 Gesture 업데이트를 거의 하지 못했습니다..
하드코딩으로 박혀있는 부분은 어차피 수정하셔야 할 것 같아서 그냥뒀어요..

그래도 몇개 고치면 플레이어로 사용할 수 있을거예요.

추가적으로 전 회사에서 썼던 UI 에셋도 함께 첨부드려요. 
동적으로 메모리 관리를 해주는 녀석이라 쓸만해요. (6만원짜리)

----------------------------------------------------------------

1.UI 호출 부분(여러분이 배우신 방법과 다를 수 있으므로 편한 방식으로 사용하시면 될 것 같아요)

GestureManager에서 기본적으로 관리가 되게 되어 있음
- 여기서 AddPanel, RemovePanel 등의 메서드로 콜백을 넘겨서 어떤 UI에 있는지 관리하게 됨
- 넘겨지는 게임오브젝트는 딕셔너리로 관리
-> 호출된 panel은 자신의 게임오브젝트와 exit 함수를 넘겨서 AddPanel에 추가

2.UI 애셋 부분(OSA)
-PanelAllGestures라고 되어 있는 게임오브젝트가 모든 제스처를 출력해주는 부분
-자식 오브젝트로 OSA 오브젝트가 있음(scrollview그냥 사용하시면 될것 같습니다.)

이 OSA 사용방법은 제가 차후에 정리해서 보내드릴게요!
정리하다보니 시간이 오래걸려서..
